# BAZIEVA Agency

## 1. Концепция (Concept)
Современный лендинг-визитка для агентства краткосрочной аренды "Bazieva".

**Текущая цель (Phase 1):**
Brand Awareness, формирование айдентики и Social Proof. Основная задача — снять нагрузку с поддержки (FAQ, инструкции) и повысить LTV через Upsell.
* **Роль сайта:** Служит "Электронным гидом" (Stage 1 плана) и визиткой для прямых продаж.
* **Ключевые метрики:** Снижение количества вопросов в чате на 90%, увеличение чека за счет доп. услуг.

**Долгосрочная цель (Phase 2+):**
Полная автоматизация и масштабирование. Превращение сайта в полноценную платформу для бронирования (Direct Booking) с собственной программой лояльности, минуя комиссии OTA. Экспансия на рынок Кавказа через управление недооцененными объектами.

## 2. Функции и страницы

### Основные разделы
* **Главная (Home):**
    * *Hero Section:* УТП, кнопка "Забронировать" (скролл к виджету), атмосферное видео/фото гор.
    * *About:* Самопрезентация владелицы, миссия ("Западный сервис на Кавказе").
    * *Social Proof:* Отзывы (интеграция или статика), ссылки на профили Airbnb/Суточно для верификации.
* **Каталог объектов (Apartments):**
    * Сетка карточек квартир.
    * Детальная страница объекта: Галерея (слайдер), список удобств (иконки), карта района.
* **Сервис и Upsell (Services):**
    * Витрина допов: Трансфер, джип-туры, завтраки, ранний заезд.
    * Call-to-Action: "Заказать в 1 клик" (переход в WhatsApp с предзаполненным сообщением).
* **Гид гостя (Guest Guide / FAQ):**
    * *Закрытый раздел (опционально) или публичный:* Пароли Wi-Fi, инструкции к технике (видео/текст), правила дома.
    * Карта: "Где поесть", "Куда сходить", маршруты.

### Интеграции
* **RealtyCalendar Widget:** Встраиваемый модуль бронирования (Iframe/Script) для проверки занятости и цен в реальном времени.
* **Messengers:** Плавающие кнопки WhatsApp/Telegram для быстрой связи.

## 3. Технический стек (Tech Stack)

Используем современный стек для мгновенной загрузки и легкости поддержки (SPA).

* **Core:** React 18 + Vite (быстрая сборка, модульность).
* **Router:** React Router v6 (навигация без перезагрузки).
* **Styling:** Tailwind CSS (быстрая верстка, адаптивность, дизайн-система).
* **UI Kit:** Headless UI или Radix UI (доступные компоненты без стилей) + Lucide React (иконки).
* **State Management:** React Context API (достаточно для текущего масштаба, Redux не нужен).
* **Hosting:** Render (автодеплой из GitHub).
* **Analytics:** Yandex.Metrica / Google Analytics (для отслеживания кликов по кнопкам брони).

## 4. Структура проекта (Project Structure)

Архитектура построена по принципу "Feature-based" с выделением данных в отдельный слой конфигурации.

```text
bazieva-agency/
├── public/
│   ├── assets/             # Статические файлы (лого, favicon, robots.txt)
│   └── images/             # Оптимизированные изображения (webp)
│       ├── apartments/     # Фото объектов (структурированы по ID)
│       └── hero/           # Баннеры
├── src/
│   ├── components/         # Переиспользуемые UI компоненты
│   │   ├── layout/         # Header, Footer, LayoutWrapper
│   │   ├── ui/             # Button, Card, Modal, Accordion (базовые "атомы")
│   │   └── widgets/        # BookingWidget (RealtyCalendar), ReviewSlider
│   ├── features/           # Компоненты, специфичные для бизнес-логики
│   │   ├── apartments/     # ApartmentCard, ApartmentList, Amenities
│   │   ├── services/       # ServiceCard, UpsellGrid
│   │   └── faq/            # FAQItem, GuideSection
│   ├── pages/              # Страницы (маршруты)
│   │   ├── Home.jsx
│   │   ├── ApartmentDetails.jsx
│   │   ├── Services.jsx
│   │   ├── GuestGuide.jsx
│   │   └── NotFound.jsx
│   ├── data/               # "Database" на клиенте (JSON/JS файлы)
│   │   ├── apartments.js   # Массив объектов (ID, name, desc, price, photos[], features[])

│   │   └── contacts.js     # Ссылки на соцсети, телефоны, гео-координаты

│   ├── App.jsx             # Роутинг
