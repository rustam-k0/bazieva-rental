# BAZIEVA Agency

## 1. Концепция (Concept)
Современный лендинг-визитка для агентства краткосрочной аренды "Bazieva".

**Текущая цель (Phase 1):**
Brand Awareness, формирование айдентики и Social Proof. Основная задача — снять нагрузку с поддержки (FAQ, инструкции) и повысить LTV через Upsell.
* **Роль сайта:** Служит "Электронным гидом" (Stage 1 плана) и визиткой для прямых продаж.
* **Ключевые метрики:** Снижение количества вопросов в чате на 90%, увеличение чека за счет доп. услуг.

**Долгосрочная цель (Phase 2+):**
Полная автоматизация и масштабирование. Превращение сайта в полноценную платформу для бронирования (Direct Booking) с собственной программой лояльности, минуя комиссии OTA. Экспансия на рынок Кавказа через управление недооцененными объектами.

---

## 2. Функции и страницы

### Основные разделы
* **Главная (Home):**
    * *Hero Section:* УТП, кнопка "Забронировать" (скролл к виджету), атмосферное видео/фото гор.
    * *About:* Самопрезентация владелицы, миссия ("Западный сервис на Кавказе").
    * *Social Proof:* Отзывы (интеграция или статика), ссылки на профили Airbnb/Суточно для верификации.
* **Каталог объектов (Apartments):**
    * Сетка карточек квартир.
    * Детальная страница объекта: Галерея (слайдер), список удобств (иконки), карта района.
* **Сервис и Upsell (Services):**
    * Витрина допов: Трансфер, джип-туры, завтраки, ранний заезд.
    * Call-to-Action: "Заказать в 1 клик" (переход в WhatsApp с предзаполненным сообщением).
* **Гид гостя (Guest Guide / FAQ):**
    * *Закрытый раздел (опционально) или публичный:* Пароли Wi-Fi, инструкции к технике (видео/текст), правила дома.
    * Карта: "Где поесть", "Куда сходить", маршруты.

### Интеграции
* **RealtyCalendar Widget:** Встраиваемый модуль бронирования (Iframe/Script) для проверки занятости и цен в реальном времени.
* **Messengers:** Плавающие кнопки WhatsApp/Telegram для быстрой связи.

## 3. Технический стек (Tech Stack)

Используем современный стек для мгновенной загрузки и легкости поддержки (SPA).

* **Core:** React 18 + Vite (быстрая сборка, модульность).
* **Router:** React Router v6 (навигация без перезагрузки).
* **Styling:** Tailwind CSS (быстрая верстка, адаптивность, дизайн-система).
* **UI Kit:** Headless UI или Radix UI (доступные компоненты без стилей) + Lucide React (иконки).
* **State Management:** React Context API (достаточно для текущего масштаба, Redux не нужен).
* **Hosting:** Render (автодеплой из GitHub).
* **Analytics:** Yandex.Metrica / Google Analytics (для отслеживания кликов по кнопкам брони).

## 4. Структура проекта (Project Structure)

Архитектура следует принципу **SSOT (Single Source of Truth)** для данных. Весь контент отделен от логики отображения и хранится в папке `data`. Стилизация централизована через конфигурацию Tailwind.

```text
frontend/
├── public/                     # Публичные статические активы (не обрабатываются Webpack/Vite)
│   ├── assets/                 # Логотипы (logo.jpg), favicon
│       └── apartments/         # Фотографии квартир (оптимизированные webp/jpg)
│
├── src/
│   ├── components/             # UI-компоненты (Presentation Layer)
│   │   ├── layout/             # Глобальные обертки
│   │   │   ├── Header.jsx      # Навигация и логотип
│   │   │   ├── Footer.jsx      # Подвал с контактами
│   │   │   └── Layout.jsx      # Основной каркас (Outlet для роутера)
│   │   └── ui/                 # Базовые элементы (Buttons, Inputs - если понадобятся)
│   │
│   ├── data/                   # Слой данных (Content Management)
│   │   └── apartments.js       # JSON-подобный массив с описанием квартир, цен, правил и фото
│   │
│   ├── features/               # Бизнес-логика, сгруппированная по сущностям
│   │   └── apartments/         # Все, что касается квартир
│   │       └── ApartmentCard.jsx # Карточка превью квартиры
│   │
│   ├── pages/                  # Маршруты приложения (Views)
│   │   ├── Home.jsx            # Главная: Hero, преимущества, избранные объекты
│   │   ├── Apartments.jsx      # Полный каталог (Grid)
│   │   ├── ApartmentDetails.jsx# Детальная страница (динамический роутинг по ID)
│   │   └── NotFound.jsx        # Страница 404
│   │
│   ├── App.jsx                 # Конфигурация маршрутизации
│   ├── main.jsx                # Точка входа (Mount point)
│   └── index.css               # Глобальные стили и импорты Tailwind
│
├── tailwind.config.js          # Дизайн-система (Цвета бренда, шрифты, отступы)
├── postcss.config.js           # Конфиг пост-процессора CSS
├── vite.config.js              # Настройки сборщика Vite
└── package.json                # Зависимости и скрипты

```

## 5. Запуск и разработка (Getting Started)

Для работы с проектом требуется установленный **Node.js** (версия 16 или выше).

### Установка зависимостей

При первом запуске выполните команду в корневой папке проекта:

```bash
npm install

```

### Запуск локального сервера

Для разработки с горячей перезагрузкой (Hot Reload):

```bash
npm run dev

```

Сайт будет доступен по адресу: `http://localhost:5173`

### Сборка для продакшена

Для создания оптимизированной версии (минификация, сжатие):

```bash
npm run build

```

Результат сборки появится в папке `dist/`.

## 6. Архитектура роутинга (Routing)

Мы используем `react-router-dom` v6. Маршрутизация настроена в файле `src/App.jsx`.

### Схема маршрутов

| URL Путь | Компонент | Описание |
| --- | --- | --- |
| `/` | `Home.jsx` | Главная страница. Содержит Hero-блок и слайдер популярных квартир. |
| `/apartments` | `Apartments.jsx` | Полный каталог всех доступных объектов. |
| `/apartments/:id` | `ApartmentDetails.jsx` | **Динамический маршрут**. `:id` берется из URL (например, `1`) и используется для поиска данных в `apartments.js`. |
| `*` | `NotFound.jsx` | Обработчик любых несуществующих адресов (404). |

**Особенность:** Все страницы обернуты в компонент `Layout.jsx`, что обеспечивает постоянное наличие `Header` (шапки) и `Footer` (подвала) при переходе между страницами.

## 7. Управление контентом (CMS-lite)

Проект не использует базы данных (Database-less), чтобы упростить хостинг и ускорить загрузку. Все данные хранятся в JavaScript файлах.

### Как изменить информацию?

1. **Данные квартир (цены, описание, фото):**
* Файл: `src/data/apartments.js` 


* Логика: Это массив объектов. Чтобы изменить цену или добавить удобство, просто отредактируйте соответствующее поле (например, `price: 3500` или массив `features`).
* *Важно:* Поле `id` должно быть уникальным числом.


2. **Цвета бренда:**
* Файл: `tailwind.config.js`
* Секция: `theme.extend.colors.brand`. Измените HEX-коды, чтобы поменять цветовую гамму всего сайта глобально.


3. **Логотип:**
* Замените файл `logo.jpg` в папке `public/assets/`. Имя файла должно остаться прежним, либо измените путь импорта в `Header.jsx`.


4. **Контакты и ссылки (WhatsApp):**
* Редактируются напрямую в компонентах `Header.jsx` (кнопка "Связаться") и `ApartmentDetails.jsx` (кнопка "Забронировать").
